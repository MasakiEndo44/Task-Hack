# Task-Hackの機能と使い方：一日のワークフロー

## 画面構成の全体像

Task-Hackは**1つの画面**にすべての情報が収まっています。画面は上から下に以下の要素で構成されています：

```
┌─────────────────────────────────────┐
│  [時計] 09:42           [ステータスバー]  │  ← 現在時刻とゾーン状況
├─────────────────────────────────────┤
│  ACT ゾーン（最大1件）                │  ← 今まさにやること
├─────────────────────────────────────┤
│  NEXT ACTION ゾーン（最大5件）         │  ← 次にやること
├─────────────────────────────────────┤
│  HOLDING ゾーン（上限なし）            │  ← 今すぐではないが把握中
├─────────────────────────────────────┤
│  CLEARED ゾーン（完了済）             │  ← 完了→アーカイブへ
├─────────────────────────────────────┤
│  [タイムライン ───────●──────────────]  │  ← 時間の流れを視覚化
└─────────────────────────────────────┘
```

## 機能一覧

### フェーズ1：基本機能（実装済み）

#### F1：ATCダッシュボード
- 4つのゾーン（ACT / NEXT ACTION / HOLDING / CLEARED）の表示
- ゾーン間のドラッグ＆ドロップによるタスク移動
- ゾーンの上限制御（ACT:1件、NEXT ACTION:5件）
- 各タスクの「フライトストリップ」カード表示
- タイムラインと現在時刻スイープライン（アニメーション）
- リアルタイムデジタル時計

#### F2：タスク管理
- タスクの作成・名前変更・削除
- タスク詳細ビュー（作業メモ、期限）
- 繰り返しタスク（毎日・毎週・毎月・カスタム間隔）の設定

#### F3：ポモドーロタイマー
- 可変長の作業セッション設定（標準25分、カスタム可能）
- セッション終了前の「ラップアップ通知」（2分前に通知）
- 休憩タイマー（5分/15分）
- タイマー終了時の通知

#### F5：ローカルJSONストレージ
- タスクデータのローカル自動保存
- アプリ再起動後もタスクが復元される
- データはローカルの `tasks.json` に保存（外部送信なし）

---

### フェーズ2：AI機能（開発予定）

#### F4：AIコプランナー（チャット機能）
- 画面下部に引き出し式のチャットUIが開く（画面遷移なし）
- 自然言語でタスクを相談できる
  - 例：「今日中に企画書を仕上げないといけないんだけど、どこから始めればいい？」
  - AIが3〜5個のサブタスクに分解し、自動でNEXT ACTIONゾーンに追加
- 画像のペースト → AI がタスクを抽出
- 「ボディダブリング」的な擬似的存在感の提供（「一緒にやろう」の感覚）

#### F6：週次AIスイープ（自動整理）
- 毎週日曜22:00（設定変更可能）に自動実行
- 完了タスクをObsidianのVaultにアーカイブ
- 週次レポート生成（「今週の達成・パターン・改善提案」）
- ユーザープロフィールの自動更新（仕事パターンの学習）

#### F7：設定拡張
- OpenAI APIキーの設定
- Obsidian Vaultのパス設定
- スイープ実行時刻の設定

#### F8：Soul.md フレームワーク
- AI秘書「Echo」の一貫した人格・振る舞いを定義するファイル
- 「絶対に守るルール」と「ユーザーが好みに合わせてカスタムできるスタイル」の2層構造

## 典型的な一日の使い方

### 朝：今日の作業を設定する（5分）

```
1. Task-Hackを開く
2. AIチャット（Echo）に向かって話す：
   「今日は午前中に週報を書いて、午後はAさんへの資料作成と
    Bさんとのミーティング準備があります」
3. Echoが自動でタスクをNEXT ACTIONゾーンに振り分ける
4. 優先順位を確認し、ドラッグ＆ドロップで微調整
5. ACTゾーンに「今すぐ始めるタスク」を1件移動
```

### 午前：作業する

```
1. ACTゾーンの1件のタスクだけを見る
2. ポモドーロタイマーを開始（例：25分）
3. タイマー終了の2分前に「ラップアップ通知」
4. 作業完了 → タスクをCLEAREDゾーンへ移動
5. NEXT ACTIONの次のタスクをACTゾーンへ移動
6. 繰り返す
```

### 途中で割り込みが来たとき

```
例：「急に上司からSlackが来た」
1. Echoチャットに「○○さんから急ぎのSlack。15分で確認と返信が必要」と入力
2. Echoが「割り込みタスク」として即時追加
3. 現在のACTタスクをNEXT ACTIONに下ろし、割り込みをACTへ
4. 対応完了後、元のタスクをACTへ戻す
```

### 夕方：今日を振り返る（2分）

```
1. CLEAREDゾーンを見る（今日完了したタスク一覧）
2. Echoに「今日を振り返って」と入力
3. Echoが「今日完了したタスク数・推定時間・次回への提案」をまとめる
4. ゾーンを空にする（翌日の準備）
```

### 週次：日曜夜に自動スイープ（自動・ユーザーの作業不要）

```
毎週日曜22:00に自動実行：
1. 完了タスクがObsidian VaultにMarkdownとして保存
2. 週次レポートが生成（「今週あなたは○件完了。得意なパターンは○○」）
3. ユーザープロフィールが自動更新
4. 翌朝、Echoが週次レポートを共有
```

## タスク管理の詳細

### フライトストリップ（タスクカード）に表示される情報

各タスクカードには以下の情報が表示されます：

| 項目 | 説明 |
|---|---|
| フライトID | 「FS0001」形式の自動採番ID |
| タスク名 | タスクのタイトル |
| 優先度 | HIGH / NORMAL / LOW |
| 推定時間 | 作業の見込み時間 |
| 繰り返し設定 | 毎日・毎週などの繰り返し |
| メモ | 詳細や補足情報 |

### ゾーンの役割と移動ルール

| ゾーン | 役割 | 上限 | 次のアクション |
|---|---|---|---|
| **ACT** | 今まさに取り組んでいる | **最大1件** | 完了→CLEARED |
| **NEXT ACTION** | すぐに取り掛かれる準備完了 | **最大5件** | ACTへ移動 |
| **HOLDING** | 把握しているが今すぐではない | 上限なし | NEXT ACTIONへ移動 |
| **CLEARED** | 完了済み（アーカイブ待ち） | 上限なし | 週次スイープでVaultへ |

- タスクはドラッグ＆ドロップでゾーン間を移動できる
- ACTゾーンがすでに1件あるときに別のタスクをACTにドロップしようとすると、アラートで制止される
- NEXT ACTIONに5件あるときも同様に制止される

### 繰り返しタスクの動作

- 繰り返し設定されたタスクが完了すると、次の実行タイミングで自動的に新しいタスクとして再生成
- 例：「メール確認」を毎日繰り返しに設定 → 毎朝自動でHOLDINGゾーンに追加
- 設定可能な繰り返し：毎日・毎週・毎月・カスタム間隔（N日ごとなど）

## ポモドーロタイマーの詳細

### タイマーの仕組み

1. タスクカードを選択してタイマー開始
2. 作業時間をカウントダウン表示
3. **ラップアップ通知**：終了2分前に「まとめ始めてください」と通知
4. タイマー終了：通知 + オプションで休憩タイマーを開始

### ラップアップ通知がある理由

ADHD傾向の人は「タイマーが鳴ったと同時に止まれない」という特性があります。作業を突然打ち切ると、未完了感や混乱が生じやすい。

2分前の通知は「今のパラグラフを書き終えよう」「今の思考を一言でメモしよう」という**安全な着地**の時間を確保するためです。

### 設定できるオプション

- 作業セッション時間（デフォルト25分、変更可能）
- ラップアップ時間（デフォルト2分）
- 休憩時間（短休憩5分 / 長休憩15分）

## インストールと起動

### 動作環境

- **Mac**：macOS対応（Apple Silicon / Intel）
- **Windows**：Windows対応（.exeインストーラー）
- **インターネット接続**：AI機能（F4, F6）使用時のみ必要。基本機能はオフライン動作

### 初回セットアップ

1. アプリをダウンロード・インストール（Mac: .dmg, Windows: .exe）
2. アプリを起動 → ATC画面が表示される
3. AI機能を使う場合：設定からOpenAI APIキーを入力
4. 週次スイープを使う場合：ObsidianのVaultパスを設定

### 通常の起動

- アプリを起動するだけで前回のタスクが自動復元される
- タスクデータはローカルに自動保存されているため、データ消失の心配なし
