# Task-Hackのプライバシーと技術仕様：安全性と信頼性

## プライバシーについて知っておくべきこと

Task-Hackは「あなたの仕事の記録」を扱うアプリです。プライバシーに関する設計方針を透明に説明します。

---

## データの保存場所

### 原則：ローカルファースト

Task-Hackのすべてのデータは**あなたのコンピューター内に保存**されます。クラウドサーバーへの送信は一切ありません（AI機能を使用する場合を除く）。

| データの種類 | 保存場所 | 外部送信 |
|---|---|---|
| タスクデータ | ローカルJSONファイル | **なし** |
| 完了タスクのアーカイブ | Obsidian Vault（ローカルMarkdown） | **なし** |
| ユーザープロフィール | ローカルMarkdownファイル | **なし** |
| AIチャットの内容 | AIリクエスト時のみOpenAIへ | あり（★後述） |
| APIキー | ローカルの設定ファイル | **なし** |

### データファイルの実体

```
コンピューター内に保存されるファイル：

Task_Hack/
└── userData/
    └── tasks.json          ← タスクの全データ

Obsidian Vault（任意のパス）/
└── 99_Task-Hack/
    ├── archives/
    │   ├── 2026-04-13_weekly.md   ← 週次アーカイブ
    │   └── ...
    ├── profile.md                  ← ユーザープロフィール
    └── soul.md                     ← EchoのSoulファイル
```

## AI機能とOpenAIへのデータ送信

### AI機能使用時のデータフロー

AI機能（Echoとのチャット・週次スイープ）を使用する際、以下のデータがOpenAI APIに送信されます：

```
送信されるデータ（例：タスク分解チャット）：
- ユーザーが入力したメッセージ
- 現在のタスクリスト（ゾーン別）
- ユーザープロフィールの要約

送信されないデータ：
- 完了済みタスクの詳細メモ（プロフィール要約のみ）
- APIキー自体（OpenAIに対してはヘッダーで渡すのみ）
```

### APIキーはユーザー自身のものを使用（BYOK方式）

Task-Hackは**BYOK（Bring Your Own Key）**方式を採用しています。

- ユーザーが自分のOpenAIアカウントでAPIキーを取得し、アプリに設定
- Task-Hackの開発者はあなたのAPIキーにもデータにも一切アクセスできない
- OpenAIのデータ利用規約はOpenAIとユーザーの間の問題（Task-Hackは関与しない）

### AI機能なしでも動く

AI機能はオプションです。APIキーを設定しなければ、AI機能は使えませんが**タスク管理・タイマー・ポモドーロなどの基本機能はすべてオフラインで動作**します。

## セキュリティの設計

### APIキーの保管

OpenAIのAPIキーはコンピューターのローカルに保存されます。アプリの設定画面から入力し、**他のアプリやサービスには共有されません**。

推奨事項：
- OpenAI管理画面でTask-Hack専用のAPIキーを作成する
- 使用量上限（Spending Limit）を設定する
- 不要になった場合はOpenAI側でキーを失効させる

### Obsidian Vaultのパス設定

週次スイープの保存先となるObsidian Vaultのパスは、ユーザーが設定します。デフォルト値は開発者のローカル環境のパスが設定されていますが、**必ず自分の環境のパスに変更**してください。

## 技術仕様

### アーキテクチャ概要

Task-Hackは**Electron**（デスクトップアプリフレームワーク）を使用して構築されています。

```
アーキテクチャの3層構造：

┌─────────────────────────────────┐
│  Renderer Process（レンダラー）  │  ← React UIが動く
│  - React 18 + TypeScript        │
│  - CSS Modules                  │
│  - ドラッグ＆ドロップ           │
├─────────────────────────────────┤
│  Preload Script（プリロード）    │  ← メインとUIの橋渡し
│  - IPC通信の安全な実装          │
├─────────────────────────────────┤
│  Main Process（メイン）         │  ← Node.js環境
│  - ファイルI/O（JSON保存）      │
│  - OpenAI API呼び出し           │
│  - スケジューラー（週次スイープ） │
│  - OS通知                       │
└─────────────────────────────────┘
```

### 使用技術スタック

| カテゴリ | 技術 | バージョン |
|---|---|---|
| デスクトップ基盤 | Electron | 最新安定版 |
| フロントエンド | React | 18 |
| 言語 | TypeScript | strict mode |
| スタイリング | CSS Modules + カスタムプロパティ | — |
| ドラッグ＆ドロップ | @dnd-kit | — |

## 週次スイープの技術的な信頼性設計

### スイープの安全性

週次スイープは完了タスクをVaultに書き込みますが、以下の順序で安全に実行されます：

```
スイープの安全実行フロー：
1. バックアップ作成（既存のarchiveファイルをバックアップ）
2. Markdownファイルへの書き込み（Obsidian Vault）
3. 書き込み確認（ファイルの存在と内容を検証）
4. JSONからの削除（確認後のみ削除）
```

**書き込みに失敗した場合は削除しない**。データ損失が起きない設計です。

### キャッチアップ機能

スイープが予定時刻に実行できなかった場合（PC電源オフ・スリープなど）の対応：

- アプリ起動時に「前回のスイープが未実行か？」をチェック
- 24時間以上経過している場合、起動時に自動でキャッチアップ実行
- スイープ完了後、次回の実行時刻をスケジューリング

## よくある疑問（FAQ）

**Q：インターネット接続は必須ですか？**
A：基本機能（タスク管理・タイマー）はオフラインで動作します。AI機能（Echoとのチャット・週次スイープ）はOpenAI APIへの接続が必要です。

**Q：データが消えることはありますか？**
A：タスクはローカルに自動保存されます。ただし、PCが壊れた・OSを再インストールした場合は消える可能性があります。Obsidian Vaultを定期的にクラウドバックアップ（iCloud、Google Drive等）することを推奨します。

**Q：複数のPC間でデータを同期できますか？**
A：現在、公式の同期機能はありません。Obsidian VaultをDropboxやiCloudの同期フォルダに置くことで、手動での同期は可能です。

**Q：OpenAIのAPIは料金がかかりますか？**
A：OpenAI APIは使用量に応じた従量課金です。Echoとのチャット1回あたり数円程度の目安です（モデルや会話の長さによります）。OpenAI管理画面で使用量の上限を設定することを推奨します。

**Q：他の人のADHD情報がEchoに送られることはありますか？**
A：Task-Hackは個人使用を前提とした設計で、ユーザー間でのデータ共有は行われません。OpenAIに送信されるのは、あなた自身の情報のみです。

**Q：アンインストールするとデータはどうなりますか？**
A：アプリをアンインストールしても、Obsidian Vaultのデータはローカルに残ります。tasks.jsonはアプリのuserDataフォルダに保存されており、アンインストール時に削除されるかOSの設定によります。
