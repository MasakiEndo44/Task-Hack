# Task-HackのAIアシスタント「Echo」：成長するあなたの外部記憶

## Echoとは誰か

**Echo（エコー）**は、Task-Hackに内蔵されているAIアシスタントです。

名前の由来は航空管制の「フォネティックコード」（無線通信で使われるアルファベットの読み方）—— E（Echo）はATCの世界で「E」を意味します。Task-HackのATC世界観にちなんで命名されています。

### Echoの役割

Echo は単なる「チャットbot」ではありません。**あなたの仕事の歴史を覚え、成長を見届けるAI秘書**です。

- タスクの分解・提案を行うコプランナー（共同計画者）
- あなたの完了タスクを整理・記録するアーキビスト（記録者）
- 週次で「あなたの成長」を言語化するレポーター
- 一緒に作業している感覚を与えるボディダブリングパートナー

---

## Echoができること：4つの機能

### 機能1：タスク分解（AIコプランナー）

**使い方**：チャット画面に「今やるべきこと・困っていること」を自然な言葉で入力する

```
ユーザー：「明日の朝に取引先へのプレゼン資料を
           仕上げないといけないんだけど、
           まだ構成も決まっていなくて…」

Echo：「わかりました。以下のタスクに分解しますね：
      1. プレゼンの目的とゴールを1文で明確化（10分）
      2. 伝えたい3つのポイントを箇条書き（15分）
      3. スライド構成のアウトライン作成（20分）
      4. 各スライドのドラフト作成（30分×3）
      5. 見直し・微調整（15分）
      NEXT ACTIONゾーンに追加してよいですか？」
```

- 曖昧な相談からタスクを自動生成
- 各タスクに推定時間を付与
- 承認してから追加（押し付けない）

### 機能2：ボディダブリング（擬似的存在感）

**ボディダブリング**とは、「他の人が近くにいるだけで集中できる」というADHD傾向の特性を活かした手法です。

Echoはタイマー開始時に「では一緒に始めましょう」「あと25分、一緒に頑張りましょう」と語りかけ、**一人ではない感覚**を提供します。

```
タイマー開始時：
Echo：「企画書の冒頭部分、一緒に取り組みましょう。
      25分後、どこまで進んだか教えてください。」

タイマー終了時：
Echo：「お疲れ様でした。どこまで進みましたか？
      次のセッションに向けて、続きの最初の一手を
      決めておきましょう。」
```

### 機能3：週次スイープ（自動整理とレポート生成）

**毎週日曜22:00**（変更可能）に自動実行されます。ユーザーが何もしなくても動きます。

#### スイープの処理フロー

```
1. CLEAREDゾーンの完了タスクを収集
2. GPT-4oで各タスクを整理・要約
3. Obsidian VaultにMarkdown形式で保存
   例：2026-04-13_weekly-archive.md
4. 週次レポートを生成
5. ユーザープロフィールを自動更新
6. 翌朝のチャットで週次レポートを提示
```

#### 週次レポートに含まれる内容

```markdown
# 週次レポート：2026年4月7日〜4月13日

## 今週の達成
- 完了タスク：23件
- 推定総作業時間：約9.5時間
- 最も集中できた時間帯：午前10〜12時

## 発見したパターン
- 午後2時以降はタスク完了率が低下（休憩が有効かもしれません）
- 「資料作成」カテゴリのタスクを最も多く完了しました

## 今週の注目ポイント
- 「Aプロジェクト企画書」を3日間かけて完成：大きな達成です
- 繰り返しタスク「メール確認」を5/5日完了：継続できています

## 来週への提案
- 火曜の午後に集中作業ブロックを設けてみませんか
- HOLDINGゾーンにある「Bさんへの提案」、今週着手できそうです
```

**レポートの設計思想**：批判ではなく発見。「できなかったこと」ではなく「できたこと」と「パターン」にフォーカスします。

### 機能4：ユーザープロフィールの自動構築

Echoは週次スイープのたびに、あなたの仕事パターンを学習・更新します。このプロフィールは時間をかけて精度が上がっていきます。

#### プロフィールに蓄積される情報

| カテゴリ | 具体的な内容 |
|---|---|
| **アイデンティティ** | 職種・役割・主なプロジェクト |
| **パターン** | 最も集中できる時間帯・苦手なタスクの種類 |
| **ゴール** | 今取り組んでいる中期的な目標 |
| **インサイト** | 過去のレポートから抽出された洞察 |

このプロフィールは、EchoがタスクをAIに送る際の「文脈情報」として使われます。EchoはあなたのことをAPIに送信された情報として「記憶」するのではなく、ローカルのMarkdownファイルに保存します。

## EchoのAI設計思想：Soul.md

### Echoには「魂」がある

Echoの一貫した振る舞いを保証するために、**Soul.md（ソウルドットエムディー）**というフレームワークが用いられています。

Soul.mdは、EchoがどんなAIであるべきかを定義した文書です。すべてのAIリクエストは、このSoul.mdを基盤として実行されます。

### 絶対に守る5つの行動原則（Behavioral Invariants）

Echoはどのようなリクエストやユーザーの指示があっても、以下の5原則を絶対に守ります：

1. **批判しない（Non-Critical）**：ユーザーのやり残し・先延ばしを批判・指摘しない。「なぜやっていないの？」とは絶対に言わない

2. **行動の小分け（Chunking）**：タスクは必ず「今すぐできる小さな一手」に分解する。「やることリストを全部終わらせよう」とは言わない

3. **達成にフォーカス（Achievement Focus）**：週次レポートは完了したことを中心に。できなかったことへの言及は最小限

4. **現在に集中（Present Focus）**：「先週は」「なぜあの時」という過去の反省より「今何ができるか」に集中する

5. **ユーザーの自律を尊重（Autonomy Respect）**：EchoはあくまでEchoの提案を「提案」する。ユーザーが承認しない限りタスクは追加しない

### Echoが言わないこと

| NG表現 | 代わりにEchoが言うこと |
|---|---|
| 「まだできていないんですか？」 | 「取り組む準備ができたら一緒に始めましょう」 |
| 「今週は成果が少なかったですね」 | 「今週完了した○件を振り返りましょう」 |
| 「もっと効率的にするべきです」 | 「このパターンが合っているみたいですね」 |
| 「全部やり終えましょう」 | 「まず最初の一手だけ決めましょう」 |

### ユーザーがカスタムできること

Soul.mdには「Coreルール（変更不可）」と「Styleガイド（ユーザーカスタム可能）」の2層があります。

**カスタム可能なスタイル例**：
- Echoの話し方のトーン（丁寧語 / フレンドリー / ビジネスライク）
- レポートの詳細度（要約版 / 詳細版）
- 提案の積極性（多めに提案 / 聞かれたときだけ提案）

## EchoとObsidianの連携

### なぜObsidianなのか

Task-Hackは完了タスクを**Obsidian**（ローカル型知識管理ツール）のVaultに保存します。

Obsidianを選んだ理由：
- すべてのデータがローカルのMarkdownファイルとして保存される（プライバシー）
- 将来的にObsidian内の他のノートとリンク・参照できる（知識の統合）
- バックアップ・移行が容易（ベンダーロックインなし）

### Vaultに保存されるデータの形式

```markdown
---
date: 2026-04-13
week: 2026-W15
tasks_completed: 5
total_estimated_time: 3h 45m
---

# 2026-04-13 完了タスク

## FS0023：週報作成
- 完了時刻：11:42
- 推定時間：60分 / 実際：75分
- メモ：先週のプロジェクト進捗の数字集めに時間がかかった

## FS0024：Aさんへのメール返信
- 完了時刻：14:05
- 推定時間：15分 / 実際：10分
...
```

このデータが蓄積されることで、Echoは「あなたの仕事の履歴書」を持つことになります。
