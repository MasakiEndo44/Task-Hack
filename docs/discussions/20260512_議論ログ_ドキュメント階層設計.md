# ドキュメント階層・分類ルール再設計ディスカッション

## 登場人物
🧑‍💼 **橘（テクニカルライター）**：ドキュメント設計の専門家。情報アーキテクチャ(IA)と目的別分類のプロ。
👩‍💻 **レオン（シニアエンジニア）**：リポジトリ管理と開発ワークフローの専門家。開発者とAIが迷わないディレクトリ構成を追求。
💅 **みさき（ギャル）**：実ユーザー（開発者目線）。「どこに何置けばいいかパッと見でわかんないと無理！」

## Turn 1｜混沌の原因と抽象的診断
🧑‍💼 **橘**: 現在のドキュメントの混沌は、「読むためのもの」「書くためのもの」「議論の残骸」がフラットに混在していることが原因です。Input/Output/Discussionという軸の提案は、情報の「ライフサイクル」に基づく抽象的診断として非常に優秀ですね。
👩‍💻 **レオン**: 具体的には、要件定義書（Output）とヒアリングメモ（Input）、SlackやAIの議論ログ（Discussion）が同じ階層にあると、先ほどの `spec-sync` のような自動化スクリプトがどれを「正典（Single Source of Truth）」として更新すべきか迷います。これは「情報の鮮度と権威の混同」という問題です。
💅 **みさき**: てかさー、そもそも名前がカオス！『20260416_解析の過去経緯に関するヒアリング.md』とかマジ長くて、後から探す気失せるんだけど〜。これってInputってことでおけ？

## Turn 2｜分類フレームワークから具体的な配置へ
🧑‍💼 **橘**: ええ。まずはInput/Output/Discussionのフレームワークを定義しましょう。
- **Input**: 外部から取り込んだ不変の一次情報（ヒアリングメモ、外部仕様書）。
- **Discussion**: 検討過程のログ（エージェントとの議論ログ、実装計画の試行錯誤）。
- **Output**: 最終的に合意された、現在「正」とすべきプロジェクトの仕様書。
👩‍💻 **レオン**: その軸の上に具体案を配置します。`docs/inputs/`、`docs/discussions/`、`docs/outputs/`（現状のoutputをoutputsに統一）としましょう。直下に置くのは README と、AI向けのメタなルール定義のみに絞ります。
💅 **みさき**: なるほどね！ Inputは「素材」、Discussionは「調理中」、Outputは「完成品」ってことっしょ！ 分かりやすっ。でもさ、ファイル名はどうするの？ 全部日付入れるの？

## Turn 3｜実例から要件原則、そして命名規則へ
🧑‍💼 **橘**: 良い実例です。みさきさんの言う「ファイル名が長い・見づらい」問題。日付プレフィックス（`YYYYMMDD_`）はInputやDiscussionなど「時系列が意味を持つ（いつ時点の情報かが重要）」なものには必須ですが、Outputは「常に最新」であるべきなので日付は不要。これを要件原則とします。
👩‍💻 **レオン**: 賛成です。つまり優先度として、Output層のファイル名は `task-hack-system-requirements.md` のようにプレーンにし、Input/Discussion層は `YYYYMMDD_topic.md` を強制します。これが運用ルールになりますね。
💅 **みさき**: Outputは常に最新版1個だけって最高！ じゃあさ、タスクごとに作る『実装計画書（plans）』とかはどうなるの？ 終わったらゴミになるけど。

## Turn 4｜一時ファイルの扱いと状態遷移の制約
🧑‍💼 **橘**: 「一時的な作業用ファイル」の扱いは IA設計 でも鬼門です。実装計画書はタスクが終われば「不要なゴミ」ではなく「過去の決定の記録（Discussion）」に移行する。情報の「状態遷移の法則」ですね。
👩‍💻 **レオン**: 具体的には、`docs/discussions/plans/` のように、Discussionの下に一時作業・計画フォルダを置くのが良いでしょう。タスク完了時（`finish-task` スキル実行時）にそこを掃除する制約を設けることで、常にクリーンに保てます。
💅 **みさき**: 天才じゃん！てことは、エージェントが `spec-sync` するときは、Outputフォルダだけを見ればいいってことだよね？ AIも迷わなくて効率爆上がり！

## Turn 5｜設計原則から明日からの具体アクションへ
🧑‍💼 **橘**: まとめましょう。設計原則は「ドキュメント・ライフサイクル分離の原則」です。
1. 素材(Input)・過程(Discussion)・正典(Output)の完全分離。
2. 正典(Output)は日付を持たず、AIの同期対象となる。
3. 過程(Discussion)は時系列で蓄積され、一時ファイルもここに内包する。
👩‍💻 **レオン**: 具体的な次の一手として、このルールを定義した `docs/AGENTS.md` （ドキュメント管理ルール）を生成しましょう。既存のルート直下にある `AGENTS.md` はプロジェクト全体のグローバルルールとし、`docs/AGENTS.md` はドキュメント管理・操作特化のルールとして分離するのが綺麗です。
💅 **みさき**: よっしゃ、これで迷子にならない！ 早くそのルールを作って、今のカオスなフォルダを整理しちゃおう！
