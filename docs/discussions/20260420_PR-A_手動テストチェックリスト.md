## FB-003-8: 週次スイープ後に Cleared タスクが消えない

### 手順

1. [X] ダッシュボードにタスクを1件以上作成し、完了ボタンを押して CLEARED ゾーンに移動させる
2. [X] CLEARED ゾーンにタスクが表示されていることを目視確認
3. [X] ステータスバー右端のスイープボタン（またはメニュー）からスイープを実行
4. [X] スイープ進捗が「collecting → generating → archiving → cleaning → done」と推移することを確認
5. [X] **「done」表示後、CLEARED ゾーンのタスクが画面から消えること** ← ここが修正ポイント
6. [X] アプリを再起動して、CLEARED タスクが復活していないことを確認

### 期待結果

| 確認項目 | 期待 |
|---|---|
| スイープ完了後の CLEARED ゾーン | 空（タスクなし） |
| 再起動後の CLEARED ゾーン | 空のまま |
| `~/.task-hack/tasks.json` の内容 | zone=CLEARED のエントリなし |
| `~/.task-hack/archive/` | `{weekLabel}.json` が生成されている |

- スイープが「done」になってもタスクが画面に残ったまま

## FB-003-7: タスク開始とタスク詳細定義のフラグが曖昧

### テスト A: 通常チャットでのステータス報告

1. [ ] AI チャット（Echo AI）を開く
2. [ ] ヘッダーに「詳細確認中」バッジが**表示されていない**ことを確認
3. [ ] 入力欄に「資料作成始めます」と送信
4. [ ] **Echo が励ましの言葉のみ返すこと（タスク JSON や TU マーカーが出ないこと）** ← ここが修正ポイント
5. [ ] 「完了しました」と送信 → 「お疲れさまでした 🛬」など労いの言葉のみ返ること

### テスト B: 「Echoに詳細を聞いてもらう」ボタン経由

1. [ ] ダッシュボードのタスクをクリックしてタスク詳細ドロワーを開く
2. [ ] 「◈ Echoに詳細を聞いてもらう」ボタンをクリック
3. [ ] Echo AI チャットが開き、**ヘッダーに「詳細確認中」バッジが表示されること** ← ここが修正ポイント
4. [ ] Echo が 5W2H に関する質問を1つ返すこと（選択肢ボタンが表示される）
5. [ ] 選択肢を選ぶ or テキストで回答し、最大3回のやり取りで確認が完了すること
6. [ ] 確認完了後、「詳細確認中」バッジが**消えること**
7. [ ] タスク詳細の「メモ」欄に回答内容が反映されること

### テスト C: 詳細確認モード中に別のチャットを送る

1. [ ] 「◈ Echoに詳細を聞いてもらう」でモードを起動
2. [ ] 「詳細確認中」バッジが表示されている状態で、別のタスクについて「来週締め切りのレポートも追加して」と入力
3. [ ] Echo の応答を確認（詳細確認モードが継続していることを確認）

### 期待結果

| 確認項目 | 期待 |
|---|---|
| 通常チャットでステータス報告 | 励ましのみ、タスク JSON・TU マーカーなし |
| 詳細確認中バッジ | ボタン押下後に表示、完了後に非表示 |
| 5W2H 確認の完了 | タスクの notes に回答が反映 |

### 失敗パターン（修正前の症状）

- 「始めます」と送ったら存在しないタスクのメモが生成される
- 詳細確認中かどうかがヘッダーで分からない
