# Task-Hack 詳細要件定義書

## 1. プロジェクト概要

「Task-Hack」は、ADHD（注意欠如・多動症）当事者向けに特化したデスクトップ型タスク管理アプリケーションです。
既存の Todo 管理ツールが抱える「完了タスクが蓄積することによる認知過負荷」や「計画立案時の麻痺（Activation Gap）」といった課題を解決するため、航空管制（ATC）のUIメタファーと、最新のAI技術（GPT-4o）による「擬似ボディ・ダブリング」および「タスクの自動構造化」を融合させました。画面上の「余白」を死守し、ユーザーの実行機能を外部から強力に支援します。

## 2. ビジネス要件

* **対象課題**: ADHD 当事者が既存のタスク管理アプリを放棄してしまう「タスク回避行動」「恥の累積」「時間盲（Time Blindness）」。
* **提供価値（Value Proposition）**: 「管理する」のではなく「着手させる」ことに特化したUI/UX。ユーザーの自己理解を深める「外部自伝メモリ」としてのAIの提供。
* **主要KPI（仮定）**:
  * デイリーアクティブ継続率（30日以上機能し続けること）
  * 1週間あたりの「CLEARED」へ移動したタスク完了数
  * ユーザーごとのプロファイル更新回数（自己理解度の指標）
* **競合優位性（7Powers）**:
  * **Cornered Resource**: ユーザー自身のパーソナルデータ（Obsidian Vault）に直接書き込み、最適化され続けるAIプロファイルの蓄積によるスイッチングコスト。
  * **Counter-Positioning**: 記録や効率を重視する既存のGTDアプリに対し、「タスクを画面から消すこと」「AIが片付けること」を主軸にしたアンチパラダイム。

## 3. ユーザー要件

* **ターゲットユーザー**: ADHD当事者、または実行機能（計画・優先順位付け・着手）に課題感を持つナレッジワーカー。
* **ペルソナ（みさきモデル）**:
  * 頻繁に新しい Todo アプリを試しては、未完了タスクの山を見て自己嫌悪に陥りアプリを開かなくなる。
  * 「今やること」以外を意識から外したいが、忘れるのも怖い。
  * 大きなタスク（例: プレゼン資料作成）を前にすると、何から手を付けていいか分からず麻痺（Task Paralysis）する。
* **MVPの定義（Phase 1~2 コア）**: 最低限のATCダッシュボード（4ゾーン）とD&Dによる物理的制約、およびタスクのローカル保存・タイマー機能を有する状態。

## 4. 機能要件詳細

### 4.1. ATC型ダッシュボード機能
* **概要**: タスクを ACTIVE, NEXT ACTION, HOLDING, CLEARED の4ゾーンで管理。
* **業務フロー（正常系）**:
  1. 新規タスクは原則「NEXT ACTION」または「HOLDING」に配置される。
  2. ユーザーは着手するタスクを「ACTIVE」にD&Dする（上限1）。
  3. タスク完了後「DONE」をクリックすると「CLEARED」へ移動する。
* **例外フラグ**: ACTIVEゾーンにすでに1つある状態でドロップした場合、操作はキャンセルされ元のゾーンにバウンスして戻る。

### 4.2. タスク詳細とタイマー機能
* **概要**: タスクをクリックして詳細な見積もり時間、サブタスクの管理を行う。ACTIVEタスクには専用タイマー（デフォルト25分等）を起動できる。
* **ユースケース**: 「集中時間」を測る際、残り2分でラップアップの穏やかな通知が鳴る。「延長」か「完了」を選択。
* **特殊要件**: タイマー切れによる「強制中断（ビープ音等の強い刺激）」は行わない（過集中保護のため）。

### 4.3. AI Co-planner (チャット・タスク自動分解)
* **概要**: 画面下部のチャットUIに自然言語または画像を投下することで、AIがタスクを生成・提案する。
* **業務フロー**:
  1. ユーザー入力：「来週の定例会議の資料作らなきゃ」
  2. AI応答：「承知しました。構成案作成、データ収集、スライド作成の3つに分解してセットしましょうか？」
  3. ユーザー承認後、NEXT ACTIONに3つのタスクカード（フライトストリップ）が生成される。
* **例外系**: オフライン時、またはAPIエラー時はチャット送信ボタンを無効化し、ローカル手動追加のみとする。

### 4.4. 擬似ボディ・ダブリングとフィードバック
* **概要**: タスク完了時のポジティブなフィードバックや、作業中のAIからのステータスバーへの「適度な声かけ」によるドパミン報酬の提供。
* **要件**: 画面操作を直接妨げない（モーダル等は使わない）トースト・または文字の微小アニメーションで表現する。

## 5. 非機能要件

* **UI/UX (Sensory Control)**:
  * **Reduce Motion**: 過度なアニメーション、無限パルスによる点滅などは撤廃または `prefers-reduced-motion` で停止。
  * **Reduce Transparency**: すりガラス等の表現は背景のノイズになるため控え、ソリッドな背景色を採用。
  * **1画面完結**: 設定画面等の例外を除き、メインのタスク操作で画面遷移（ページ戻り）を発生させないこと。
* **パフォーマンス**: D&D操作は100ms以内で追従すること（`dnd-kit`を使用）。Electron アプリケーションの起動は3秒以内。
* **データ可用性とセキュリティ**: 全てのタスクデータはローカルのJSON等に即時保存され、クラウドデータベースへの送信は行わない（AI APIへのプロンプト通信のみ）。

## 6. インテグレーション要件

1. **OpenAI API**:
   * **用途**: 自然言語テキスト解析、画像（Vision）解析によるタスク生成、ユーザープロファイルへの要約。
   * **連携方式**: Electron Mainプロセスから `openai` Node SDK または REST 経由で呼び出し。
2. **Obsidian Vault (Local File System)**:
   * **用途**: 週次レポートのMarkdown書き出し、ユーザーの Identity・Pattern 分析用Markdown読み込み。
   * **連携方式**: Node.js `fs` モジュール等による直接I/O。設定画面でVaultの絶対パスを指定させる必要がある。

## 7. 技術選定とアーキテクチャ

* **基盤**: Electron, Vite (`electron-vite` ボイラープレート)
* **フロントエンド**: React 18, TypeScript, CSS Modules (Vanilla CSS)
* **状態管理**: React `useReducer` による一元管理。D&Dには `@dnd-kit/core` および `sortable` を利用。
* **データ永続化**: Node.js `fs/promises` または `electron-store` によるローカルJSON管理。
* **アーキテクチャ方針**: メインプロセス（ファイルI/O、LLM API通信）とレンダラープロセス（React UI）を `preload.ts` による IPC 通信で完全に分離し、セキュアな設計とする。

## 8. 開発プロセス / スケジュール

* **Phase 1 (MVP Core)**: UIレイアウト、フライトストリップ、D&Dのコアロジック実装。（完了）
* **Phase 2 (Data & Timer)**: 永続化、ポモドーロタイマー、詳細画面の実装。
* **Phase 3 (AI Chat)**: 画面下部ドロワーUI、OpenAI インテグレーション。
* **Phase 4 (AI Growth)**: Obsidian Vault連携、週次レポート生成バッチ処理、タスクの自動アーカイブ（CLEAREDゾーンから消去）。

## 9. リスクと課題

* **AIプロンプトの不確実性**: タスク分解やプロファイル作成などのAIによる出力が、ADHDユーザーにとって有益な構造にならないリスク。
  * *対策*: プロンプトエンジニアリングの徹底および検証。提案されたタスクは必ずユーザーの「承認」を経てボードに追加するフェーズを設ける。
* **Vault パス変更時の対応**: ユーザーが Obsidian Vault の場所を移動した際のエラーハンドリング。
  * *対策*: アプリ起動時およびレポート生成時にパスの生存確認を行い、見つからない場合はアラートと設定画面への誘導を出す。

## 10. ランニング費用と運用方針

* **インフラ費用**: ゼロ（デスクトップアプリとして配布するためサーバー代は不要）。
* **API費用（通信費）**: ユーザー自身が自前の OpenAI API Key を提供（Bring Your Own Key）することを想定。個人利用であれば月額 $1〜$5 程度（仮定）。
* **運用体制**: 開発者自身のドッグフーディングを中心としたローカル運用から開始。
